## Task 1 — Policy documents

**Files:** `docs/security/sensitive-content-handling.md`, `docs/security/artifact-retention.md`

**Action:**

1. **Sensitive content handling:** Define default sensitivity for analyzed code, generated markdown, `_state` JSON, hooks/transcripts; operator duties; link pattern catalog to script (same categories as helper).
2. **Artifact retention:** Describe typical local artifacts (vault, caches), gitignore relationship, recommended cleanup cadence, explicit **no default telemetry** (NFR-1).

**Verification:** `test -f` both paths.

---

## Task 2 — Redaction helper

**Files:** `scripts/redact_public_text.py`

**Action:**

1. Stdin → stdout filter; non-zero on unexpected read failure only (normal empty ok).
2. Regex patterns: Bearer/JWT-like segments, `sk-` prefix keys, AWS access-key-shaped tokens, email-shaped substrings, long repeating hex (optional conservative threshold).
3. Replace matches with `[REDACTED]`; document that false positives are acceptable for hook stdout.
4. `chmod +x`; `python3 -m py_compile` passes.

## Task 3 — Hook integration

**Files:** `skills/code-to-docs-hooks/hooks/digest-on-start.sh`

**Action:**

2. Pipe `PROJECT`, `MODULES`, and the `git diff --stat` summary line through `redact_public_text.py`.
3. Keep existing basename vault display and exit codes.
